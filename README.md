## Ответы на вопросы

### 1. Чем Minimal API отличается от MVC в ASP.NET Core?

Minimal API - Минимум кода Подходит для простых мршрутов и микросервисов Быстрее в разработке

MVC - Чёткое разделение на Model, View, Controller Подходит для крупных приложений Более сложная архитектура

### 2. Что произойдёт, если поставить `app.MapGet(...)` до `app.Use(...)`?

для middleware важен порядок, поэтому важно все писать последовательно

Если сделать это до то запрос может обработаться маршрутом и middleware не выполнится

### 3. Почему ASP.NET Core преобразует PascalCase в camelCase при сериализации JSON?

По умолчанию используется System.Text.Json который делается по стандартам JavaScript, в JS принято использовать camelCase

### 4. Что означает код ответа HTTP 401? А 404? А 200?
200 OK — запрос успешно выполнен
401 Unauthorized — требуется авторизация
404 Not Found — ресурс не найден

### 5. Чем dotnet run отличается от dotnet watch run?

dotnet run просто запускает приложение один раз
dotnet watch run отслеживает изменения в коде, автоматически перезапускает приложение при изменениях
